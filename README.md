## Core Capabilities

### ūüŹõÔłŹ AI Company Governance

Unlike traditional AI agents that operate as stateless chat interfaces, Dong AI implements a **full corporate decision-making pipeline**: each project undergoes structured debate, execution is distributed across dynamically assembled specialist workers, and every phase exits through a quality gate enforced by board-level scoring.

```
User Request ‚Üí CEO ‚Üí Red/Blue Debate ‚Üí Design + Requirements
    ‚Üí Worker Pool (cross-review + automated tests)
    ‚Üí Board Review ‚Üí Phase Gate (‚Č• 6.0/10)
    ‚Üí Next Phase or Terminate
```

### ūüß† Graph Memory

Conventional LLM applications lose context beyond their window limit. Dong AI's **graph memory layer** persists structured knowledge across sessions:

- **Symbol indexing**: Every function, class, interface, and dependency is automatically extracted and stored in a queryable graph database
- **Contextual injection**: When starting a new task, the system queries the graph for relevant symbols, signatures, and dependency relationships ‚ÄĒ delivering precise context instead of raw text
- **Cross-phase coherence**: Phase 3 can reference symbols defined in Phase 1 with exact signatures, not vague descriptions
- **Requirement traceability**: Each task's output is linked back to specific design requirements, enabling coverage analysis and quality scoring

This architecture supports **coherent development across large, multi-phase projects** ‚ÄĒ not by stuffing text into a context window, but by maintaining a structured, queryable knowledge graph.

### ūüďč Dynamic Project Pipeline

The CEO automatically identifies project type and generates a custom execution pipeline via LLM. No hardcoded workflows ‚ÄĒ every project gets a tailored plan.

| Input | Detection | Generated Pipeline |
|-------|-----------|-------------------|
| "Build a config system" | software | Scaffold ‚Üí Core ‚Üí Test ‚Üí Release |
| "Write a cyberpunk novel" | novel | World-building ‚Üí Characters ‚Üí Chapters ‚Üí Revision |
| "Develop a pixel RPG" | game | Design doc ‚Üí Mechanics ‚Üí Content ‚Üí Build |
| "Analyze this architecture" | analysis | Collection ‚Üí Analysis ‚Üí Report |
| "Audit this codebase" | audit | Scope ‚Üí Review ‚Üí Findings |

Each phase is executed by **dynamically recruited workers** (generated by LLM based on task requirements), then undergoes **cross-review**, **automated testing**, and **board scoring** with a minimum quality gate of 6.0/10.

## Quick Start

```bash
# Installation
pip install dong-ai
pip install 'dong-ai[all]'     # full dependencies incl. API server

# Interactive setup wizard ‚ÄĒ detects hardware, selects mode, configures context
dong setup

# Start interactive TUI
dong chat

# One-click project execution
dong run "Build a configuration management system"

# Start OpenAI-compatible API server ‚Üí http://localhost:8648
dong serve
```

### Command Reference

```
dong chat          Interactive TUI          dong config      Manage configuration
dong run "req"     One-click execution       dong skill       List/create skills
dong serve         API server                dong session     View chat history
dong setup         Setup wizard              dong mcp         Discover MCP tools
dong detect        Detect available models   dong cron        Scheduled tasks
dong version       Version info              dong webhook     Webhook management
```

## Architecture

```
User Layer:       dong chat / dong run / dong serve / API clients

Orchestration:    CEO ‚Üí DesignEngine(Red/Blue debate) ‚Üí WorkerPool(self-heal, cross-review)

Engine Layer:     ModelPool(20+ providers, auto failover) ‚Üí LLMClient(unified HTTP/SSE)

Storage Layer:    Datastore(SQLite)
                  ‚Ēú‚ĒÄ‚ĒÄ MemoryRepository      Fact KV
                  ‚Ēú‚ĒÄ‚ĒÄ SessionRepository     Conversation history
                  ‚Ēú‚ĒÄ‚ĒÄ ProjectRepository     Decisions & module states
                  ‚Ēú‚ĒÄ‚ĒÄ LoreRepository        World-building (novel mode)
                  ‚ĒĒ‚ĒÄ‚ĒÄ GraphRepository       Code symbols, dependencies, requirements trace
```
